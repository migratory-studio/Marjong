# AI Mahjong — ディレクション指針

このリポジトリで作業するときは、以下の「何を重視しているか」を最優先の判断軸にすること。
コードの綺麗さや麻雀ロジックの正しさは前提。その上で **下記の体験目標に資するか** で実装の良し悪しを判断する。

## このゲームの核（最重要）

麻雀に新しい体感をもたせること、そして何より **キャラクターと自身（プレイヤー）の距離感・キャラへの愛着** を大事にする。
ディレクションの主戦場は **ナラティブと体感設計**。機能を足すときは必ず「これはキャラへの愛着 / 体感に効くか？」を自問する。

- 麻雀部分は「点棒＝HP」を軸にした体感づくり（孤独な麻雀を“共闘”に変える）。
- キャラは立ち絵＋セリフで生きる。正典は `scenario-forge/reference/world.md`（九蓮宝士 世界観）。
- 現状の主役は **詩玥（シ・ユエ）**。中華娘口調・自称「我（ウォ）」・口癖「ツモれば勝ち」。
  表は楽天/軽口、素性（深謀遠慮＝読みの達人だった過去／恩師の喪失／点棒嫌い＝自罰）は段階的に滲ませる。

## 愛着が生まれる原理（設計の物差し）

> **蓄積（履歴がある）× 固有性（"私"を見ている）× 双方向（返せる）× 反転（意味が変わる）**

新しい演出・セリフ・システムは、この4要素のどれを強めるかを意識して設計する。

## 具体ピラー（実装の指針／推し順）

1. **関係性の「状態」＝ 蓄積の可視化** … `cond.bondMin`（絆Lv下限）を `condMatches` に足し、同一eventのセリフを段階解放。
   ※ ゲージ等で**数値を見せない**。「最近言い方が変わった」とプレイヤーが自分で気づくのが最高の報酬。口調が崩れる瞬間は最高絆の特別報酬。
2. **「あなたを覚えている」＝ 固有性** … `ctx` に軽量なプレイヤー履歴（連勝・前局結果・多用する打ち筋）を積み、セリフから参照する。
3. **対局中の共闘相槌＝ 共在感** … 得点イベントだけでなく、テンパイ/リーチ等の局中マイクロ反応を発火。立ち絵を「相棒」にする。
4. **素が滲む瞬間を“設計された階段”に** … pinch/bottom の素出しを、絆＋シナリオ進行で段階開示。**点棒嫌い（HP＝点棒なのに点棒を憎む）はメカ×物語の融合点**として活かす。
5. **双方向＝ プレイヤーが返せる** … pinch時や対局後に短い2択を返せ、キャラがそれを覚える。
6. **口癖の反転** … 「ツモれば勝ち」を同じ言葉のまま文脈で意味を変える（軽口→恩師の言葉→呪い→解放）。

## 作業スタイル（モデルの使い分け）

メインセッション（Fable 5）は **設計・プランニング・デザイン議論・ディレクション判断・監査・レビュー** に専念し、トークンを多く食う作業はサブエージェントに切り出してコストとコンテキストを節約する。

- **サブエージェントに切り出してよいもの**（`model: sonnet` を基本、難しめは `opus`）
  … コードベースの探索・調査、仕様が確定した後の定型実装・リファクタ、テスト追加、データ表（マスタ類）の機械的編集、grep的な横断確認。
- **メインセッション（Fable）で直接やるもの**
  … 設計・プランニングの議論、レビューと最終判断、実装難易度が特に高い箇所、そして **キャラのセリフ・ナラティブの文言**。詩玥の口調（中華娘・我（ウォ）・ツモれば勝ち）や世界観に関わるテキストはサブエージェントに丸投げしない。
- サブエージェントは会話の文脈を引き継がない。委譲するときは対象ファイル・確定済み仕様・関係する上記ピラーをプロンプトに自己完結で書くこと。

## セリフ・システム（既存実装の要点）

- マスタ駆動。`src/data/characterVoiceMaster.js`（文言）＋ `src/data/voiceLines.js`（解決ロジック `pickVoiceLine`）。
- event: `matchStart` / `agari` / `damage` / `matchEnd`。tier条件で分岐し、一致候補からランダム選択。
- 拡張は `condMatches` に条件キーを1行足すだけ（`skillLevelMin` が前例。`bondMin` 等も同パターンで足せる）。
- 詩玥のみ本実装。他キャラは `templateLines` で自動生成（未記入は `［テンプレ］` でgrep可）。
