# オンライン一時クローズ（メンテナンス/おやすみ）設計

通信対戦を**ゲーム本体を作り直さずに**一時停止できる仕組み。コスト管理・不具合時の緊急停止・
メンテ・「テスト中だから一旦閉じる」等に使う。**キャラ愛着を切らさない見せ方**（CLAUDE.md の核）を
最優先に、無機質なエラーではなく「詩玥が“卓を整えてる”」という在り方で見せる。

---

## 1. 状態（4つ）

| 状態 | 意味 | 新規入室 | 進行中の対局 |
|---|---|---|---|
| **OPEN** | 通常営業 | 可 | 継続 |
| **CLOSED** | 計画的おやすみ/メンテ（運営が閉じた） | **不可**（在キャラ案内＋復帰予定） | **継続**（途中で蹴らない） |
| **DRAINING**（任意・v2） | 新規だけ止めてメンテ準備 | 不可 | 継続 | 
| **UNREACHABLE** | サーバ応答無し（落ちてる/ネット不調） | 不可（接続確認できません＋再試行） | – |

v1 は **OPEN / CLOSED / UNREACHABLE** の3つで十分（DRAINING は CLOSED に畳む＝「新規不可・進行中は継続」が既にその挙動）。

---

## 2. 仕組み（ゲーム本体は再デプロイ不要）

トグルは**サーバ側（Cloudflare Worker）**に置く。ゲーム本体（GitHub Pages）は触らない。

### Worker 側（`worker/index.js`）
- `GET /status` → JSON を返す（**CORS 必須**：ゲームは github.io、Worker は workers.dev＝別オリジン。`Access-Control-Allow-Origin: *` を付ける）。
  ```json
  { "open": true, "message": "", "until": null }
  ```
- フラグの置き場（簡単→堅牢の順）：
  1. **Worker 環境変数 `ONLINE_OPEN` / `ONLINE_MESSAGE`**（v1 推奨）。Cloudflare ダッシュボード → Worker → Settings → Variables で編集＝**Worker だけ即再デプロイ**（ゲーム本体は無関係）。`wrangler deploy` でも可。
  2. （v2）**KV** に置けば**再デプロイ無し**でダッシュボードから値を書き換えるだけ。トグル頻度が増えたら移行。
- `/ws`（新規接続）：閉鎖中は**アップグレードを拒否**（503 か即クローズ）。`/status` を見ずに直接来たクライアントへの保険。
- **進行中の対局は無傷**：閉鎖は「新規 DO 接続を断る」だけ。既存 DO の卓は最後まで回る（途中で人を蹴らない）。

### トグル操作（運営＝あなた）
- **閉じる**：ダッシュボードで `ONLINE_OPEN=false`（必要なら `ONLINE_MESSAGE="メンテ中。6/15 12:00 まで"`）→ 保存（Worker 自動再デプロイ）。
- **開ける**：`ONLINE_OPEN=true`（or 変数削除）。
- 反映は数十秒。ゲーム本体は一切触らない。

---

## 3. クライアントの挙動

- **接続先 URL から status URL を導出**：`wss://…/ws` → `https://…/status`（定数 1 つ）。
- **通信対戦に入った時に `/status` を取得**（ホームでは取らない＝起動を軽く保つ）。短時間キャッシュ（例 30 秒）。
  - `open` → 通常（ルーム/マッチング有効）。
  - `!open` → **おやすみ画面**を表示し、対局開始を無効化。
- **接続失敗の保険**：`/status` が OPEN でも実際の `/ws` 接続が落ちた/拒否されたら、UNREACHABLE 扱いの案内に切替（既存の connect 失敗ハンドリングを画面内通知へ格上げ）。
- **対局中に運営が閉じても**、その対局は続行（サーバが既存卓を維持）。次の入室から閉鎖が効く。

## 4. ユーザーへの見え方（UX・ここが本題）

### 4-1. ホーム（任意）
- 既定は**変更なし**（status をホームで取らない）。
- （任意・後で）軽量バッジ：`通信対戦` ボタンに小さく `おやすみ中` ピル。やるなら status をホームでも 1 回だけ取得しキャッシュ。まずは無しでOK。

### 4-2. 「通信対戦」画面（モード選択）＝おやすみ時はここを差し替え
無機質な「メンテナンス中」ではなく、**詩玥の立ち絵＋セリフ**で見せる（在キャラ・愛着優先）。

```
┌───────────────────────────────────────────┐
│  通信対戦   [テスト中]                          │
│                                              │
│   ┌────────┐   通信対戦は ただいま おやすみ中     │
│   │ 詩玥    │   「我(ウォ)の卓、今ちょっと整えてる    │
│   │ 立ち絵  │    トコ。すぐ戻るから……              │
│   └────────┘    ツモれば勝ち、また一緒にやるヨ！」   │
│                                              │
│   〔運営メッセージ：6/15 12:00 ごろ再開予定〕      │
│                                              │
│   [ 詩玥とCPU練習する ]   [ ← ホームへ ]         │
└───────────────────────────────────────────┘
```

- **見出し**：「通信対戦は ただいま おやすみ中」（“メンテナンス”より柔らかく）。
- **詩玥のセリフ（在キャラ）案**（口調：中華娘・自称「我(ウォ)」・口癖「ツモれば勝ち」）：
  - A（軽口）：「我(ウォ)の卓、今ちょっと整えてるトコ。すぐ戻るから……ツモれば勝ち、また一緒にやるヨ！」
  - B（短）：「ごめんネ、今は店じまい中アル。少し待ってネ。」
  - C（素が滲む高絆版・任意）：「……たまには卓を磨く時間も要るのヨ。すぐ戻る。待っててくれる？」
  - ※将来 voiceLines に `event: onlineClosed` を足せばマスタ駆動・絆段階で出し分け可（pickVoiceLine 既存パターン）。
- **運営メッセージ**：`ONLINE_MESSAGE`（再開目安など）。空なら省略。
- **フォールバック導線（重要）**：行き止まりにしない。
  - **「詩玥とCPU練習する」** → 既存の**ループバック（サーバ不要）online**を起動（`startOnlineMatch` / `window.__ONLINE_WS_URL="loopback"` 相当）。サーバが閉じてても遊べる＝愛着の連続性。
  - 「← ホームへ」。

### 4-3. 接続できない時（UNREACHABLE）
- 文面だけ変える：「サーバに接続できなかったみたい。通信状況を確かめて、もう一度試してネ。」＋ **[ もう一度 ]**（再 fetch/再接続）＋ **[ 詩玥とCPU練習する ]**。
- “おやすみ（意図的）”と“繋がらない（不調）”は文面を分け、どちらも CPU 練習へ逃がす。

### 4-4. トーン指針
- 「障害/エラー」ではなく「**ちょっとお休み**」。プレイヤーを責めず、待つ理由を**キャラの口で**伝える。
- 復帰予定があれば必ず添える（不安を減らす）。
- どの状態でも**詩玥と打てる道（CPU練習）を必ず残す**。

---

## 5. 実装スコープ（着手時の見積り）

- **Worker**：`/status`（CORS付き）＋ `/ws` の閉鎖ガード＋ `ONLINE_OPEN`/`ONLINE_MESSAGE` 参照（小）。
- **クライアント**：status fetch＋おやすみ画面（`onlineScreen` 差し替え or 新カード）＋ CPU練習フォールバック配線＋ UNREACHABLE 文面（中）。
- **セリフ**：詩玥の onlineClosed セリフ（私が書く・在キャラ）。将来マスタ化は任意。
- **運営手順**：`docs/online-deploy-setup.md` に「閉じ方/開け方（環境変数トグル）」を追記。

> v1 は **Worker 環境変数トグル**で十分。トグルを頻繁にやるなら v2 で **KV 化（再デプロイ無し）**。
